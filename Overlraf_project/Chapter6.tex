\chapter{Future work}

One important extension of the present research is the development of a comprehensive deployment guideline framework for semiconductor defect classification systems operating in both edge and cloud environments. In the next phase of this work, a detailed guideline document will be prepared to support deployment decisions based on application requirements.

The proposed work will compare edge and cloud deployment scenarios using critical factors such as inference memory constraints,latency, computational resource availability, scalability, privacy considerations, operational cost, and maintenance complexity. This will assist semiconductor manufacturers in selecting the most suitable deployment architecture for specific inspection workflows.For example, latency-sensitive inline inspection tasks may benefit from edge deployment, whereas large-scale historical analytics and centralized model retraining may be more suitable for cloud environments. The future study will also investigate hybrid edge-cloud architectures combining real-time local inference with centralized intelligence.
